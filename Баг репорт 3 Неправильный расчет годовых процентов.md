# Описание.
Метод yearChange неверно рассчитывает проценты для отрицательного баланса.
# Локация дефекта:
        
https://github.com/netology-code/javaqa-team-diplom-2/blob/7e01d1602ba0e7787782ab0ac6d54e4d04a0db01/src/main/java/ru/netology/javaqadiplom/CreditAccount.java#L82C1-L84C3

# Шаги воспроизведения:

 
1. Создать новый кредитный счет с начальным балансом 0 рублей и кредитным лимитом 5000 рублей, процентная ставка: 15% годовых
1. Оплатить 2000 рублей (на счету: -2000 рублей, долг банку)
1. Посчитать проценты за год
1. Проверить результат операции и баланс после оплаты

# Тест для проверки:

    @Test
    void shouldCalculateYearChangeForNegativeBalance() {
    CreditAccount account = new CreditAccount(0, 5000, 15);
    account.pay(2000);
    
    int yearChange = account.yearChange();
    
    assertEquals(-300, yearChange);}

*Ожидаемый результат:* Программа должна сказать: -300 рублей

*Фактический результат:* Программа говорит: 0 рублей
